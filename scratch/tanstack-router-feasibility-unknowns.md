## Empirical resolver

The feasibility gate is CLOSED. The spike establishes a hard requirement for any later work: use code-based TanStack routes inside a client-only router host (or separately design a proper isomorphic router); do not assume the browser-created router can SSR directly through current Vinext. It deliberately did not prove the user-facing Week/Day navigation contract; that is the first delivery slice's acceptance work, with the already chosen browser cases in Rows 3–5.
